# P6 App 層一発置き換え 実機手動チェックリスト

**対象**: P6(App 層一発置き換え + Scintilla/v1 系一括撤去)完了時点の App 層全機能。P7(実機総合検証・撤去・リリース整備)へ進む前の中間検証。SR は NVDA / PC-Talker / ナレーター(汎用UIA)の 3 種で確認。SR 未起動時も基本操作を確認。

**前提**:
- worktree: `<repo>\.worktrees\custom-editcontrol-design`(ブランチ `feature/custom-editcontrol-design`)
- ビルド: `dotnet build yEdit.sln -c Release`
- 起動: `dotnet run --project src/yEdit.App -c Release` or Release ビルドの yEdit.exe を起動
- 実機フルマトリクスは P7 で改めて実施(ここは中間確認・許容範囲判断)

**判定**:
- OK: 期待動作に一致
- 許容: 完全一致ではないが実用上問題なし・SR 依存の細部差異
- NG: 期待動作を満たさない → Critical/Important の指摘

**表記**: `[  ] 項目名` にチェックを入れて結果を記録。各項目末尾に SR 別備考を追記可能。

## A. 起動・ドキュメント基本

- [ ] A-1 起動時に無題タブ 1 つが開く(空バッファ・キャレット位置 0)
- [ ] A-2 タイトルバー "yEdit" 表示・カーソル行/桁が下部ステータスバーに表示
- [ ] A-3 「編集領域」に自動でフォーカスが当たっている(即座にタイプできる)
- [ ] A-4 SR がタブ切替時にタイトル(ファイル名+位置=タブラベル)を読む(NVDA/PC-Talker/ナレーター全部)

## B. ファイル読み込み(3 エンコーディング + BOM/改行検出)

- [ ] B-1 UTF-8(BOM なし)を開く: 本文正常表示・エンコーディング/改行がステータス正しく表示
- [ ] B-2 UTF-8(BOM 付き)を開く: BOM 状態がステータスに反映
- [ ] B-3 Shift_JIS(CP932)を開く: 全角/半角/英数字が正しくデコード
- [ ] B-4 EUC-JP(CP51932)を開く: 3 バイト文字を含んでも正しくデコード
- [ ] B-5 100MB 超の UTF-8 プレーンテキストを開く: OOM せずロード完了(Stream I/O 経路)
- [ ] B-6 CRLF/LF/CR の 3 種改行を含むファイル: 検出結果(推定 EOL)がステータス表示・編集領域では改行として認識

## C. 保存・上書き

- [ ] C-1 `Ctrl+S` で上書き保存(既存ファイル): 変更マーク "*" が消える
- [ ] C-2 `Ctrl+Shift+S` で名前を付けて保存: エンコーディング/BOM/改行 の選択ダイアログが出る
- [ ] C-3 名前付き保存後にファイル名がタブラベルに反映
- [ ] C-4 保存中 UI がフリーズしない(バックグラウンド I/O)
