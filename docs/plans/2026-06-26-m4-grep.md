# M4 複数ファイル横断 grep — 実装計画

**Goal:** フォルダ再帰 grep を、SR が結果一覧を読み・該当へジャンプできる形で実装する。Core は SR 非依存で単体テスト可能に保つ。

**Architecture:** `yEdit.Core.Search.GrepService`（純ロジック・`TextSearcher`/`TextFileService` 再利用）＋ `yEdit.App` の `GrepController`/`GrepDialog`/`GrepResultsWindow`（モードレス・別スレッド実行・ListBox 結果）。設計根拠: `docs/plans/2026-06-26-m4-grep-design.md`。

**Tech Stack:** .NET 9 / C# / WinForms / Scintilla5.NET / `System.Text.RegularExpressions` / xUnit。

## Task 3: Core — GrepService（列挙・glob・行照合・キャンセル・エラー集約）
**Files:** `src/yEdit.Core/Search/GrepService.cs`
- 安全な深さ優先列挙（不可ディレクトリはスキップしエラー集約）。
- glob → 正規表現翻訳（`*`/`?`・`IgnoreCase`・複数パターン・空＝全件）。
- バイナリ（NUL）スキップ。`DecodeBytes` で復号。行分割＋絶対オフセット積算＋行先頭マッチ。
- 協調キャンセル（部分結果＋`Cancelled`）。ファイル毎 I/O / Regex タイムアウトはエラー集約し継続。
- 進捗 `Report`。
ビルド確認 → コミット。
